## Executive Summary

The current research state is more skeptical than the previous V5 evidence log.
V5 correctly moved toward strict fold-local scaler, encoder, classifier, replay,
and Lumibot-style artifacts, but the audit found one remaining V5 candle feature
leak inherited from the old candle research path:

- `src/v5/candle_oos.py:add_candle_extra_features()` used left-labeled 1H/4H
  resampling for EMA features.
- A current M15 row could therefore receive higher-timeframe values affected by
  later M15 closes inside the same 1H or 4H bin.
- This is the same class of multi-timeframe look-ahead that v4 identified as a
  reason old candle and hybrid Sharpe claims collapsed.

As of this update, the V5 code path has a regression test for that leak and the
resampling now labels higher-timeframe closes at completed bar times. All V5
candle OOS artifacts generated before this fix are research-only and must be
regenerated before any Sharpe claim is trusted.

The first regenerated USDJPY enc30 candle-trail run after the fix failed:
`usdjpy-candle-trail-v5-post-mtf-fix-enc30` produced Sharpe `-1.188`, daily
Sharpe `-0.931`, return `-0.436%`, 421 trades, and win rate `41.3%`. The timing
proof passed, so the negative result is the current best evidence, not a tooling
failure.

## Code Audit Finding: MTF Candle Feature Leakage

Root cause:

```python
close_1h = df_raw["close"].resample("1h").last().ffill()
ema_1h_m15 = ema_1h.reindex(df_raw.index, method="ffill")
```

With left-labeled resampling, a 1H bin labeled `01:00` can contain the `01:45`
M15 close. Forward-filling that value onto `01:15` gives the model information
from the future. The 4H path had the same issue with a larger look-ahead window.

Fix:

```python
close.resample(rule, label="right", closed="right").last().ffill()
```

The fix labels higher-timeframe closes at the completion time of the higher bar.
The new regression test mutates future M15 bars inside the same 1H/4H bin and
requires the current feature row to remain unchanged.
